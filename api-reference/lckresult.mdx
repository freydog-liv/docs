---
title: "LckResult"
description: "Represent the outcome of operations in the LIV and provide a consistent way to check for success or failure, capture error codes, and retrieve result values."
---

`LckResult` is used across all LIV Capture Kit (LCK) APIs to ensure operations are safe to call and provide error feedback instead of throwing exceptions.\
When successful, `LckResult<T>` may also carry a typed result (`Result`).

Developers should always check the `Success` property before accessing `Result` to avoid errors.

## Usage

```c#
var result = LckService.GetService();
if (result.Success)
{
    LckService service = result.Result;
    Debug.Log("Service retrieved successfully");
}
else
{
    Debug.LogError($"Failed: {result.Error} - {result.Message}");
}
```

## References

| Name      | Type        | Description                                                                      |
| :-------- | :---------- | :------------------------------------------------------------------------------- |
| `Success` | `bool`      | Indicates whether the operation was successful.                                  |
| `Message` | `string`    | Provides an optional message, typically used for error information.              |
| `Error`   | `LckError?` | Contains the error code or type if the operation failed.                         |
| `Result`  | `T`         | The result value of the operation if successful; default value of `T` otherwise. |
